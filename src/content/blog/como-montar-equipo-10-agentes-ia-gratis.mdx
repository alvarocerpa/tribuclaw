## El equipo

Mi equipo se llama así: Kimo, Mao, Pluma, Mika, Mei, Data y Rex. Cada uno tiene un trabajo específico.

```
         ┌─────────────────────────┐
         │       KIMO (CEO)        │
         │   Coordina y delega     │
         └──────────┬──────────────┘
                    │
        ┌───────────┼───────────┐
        │           │           │
   ┌────┴───┐ ┌────┴───┐ ┌────┴───┐
   │  MAO   │ │ PLUMA  │ │  REX   │
   │ Código │ │  Copy  │ │  SEO   │
   └────────┘ └────────┘ └────────┘
        │           │
   ┌────┴───┐ ┌────┴───┐ ┌────────┐
   │ MIKA   │ │  MEI   │ │  DATA  │
   │Diseño  │ │ Admin  │ │Datos   │
   └────────┘ └────────┘ └────────┘
```

**Kimo (CEO / Coordinador):** Recibe mis peticiones, decide quién hace qué, supervisa la calidad. Modelo: GLM-5.

**Mao (Código):** Escribe código, revisa PRs, mantiene el repo. Modelo: GLM-5 Turbo.

**Pluma (Copy):** Escribe posts, emails, landing pages. Todo lo que sea palabras, pasa por ella. Modelo: GLM-5.1.

**Mika (Diseño):** Genera miniaturas, imágenes para posts, assets visuales. Modelo: Gemini.

**Mei (Admin):** Gestiona tareas, organiza el tablero (Paperclip), mantiene el orden. Modelo: GLM-5.

**Data (Analítica):** Analiza métricas, SEO, rendimiento. Modelo: ZAI.

**Rex (SEO):** Optimiza posts, posiciona en Google, vigila rankings. Modelo: ZAI.

## El flujo de trabajo

Cuando quiero algo, le hablo a Kimo. Nunca a los demás directamente. Así funciona:

**1. Yo le digo a Kimo:** "Necesito un post sobre automatización con agentes IA para el blog."

**2. Kimo delega:**
- Pluma: escribe el post
- Rex: optimiza SEO (título, meta description, keywords)
- Mika: genera la miniatura

**3. Cada agente ejecuta su parte.** Pluma escribe, Rex optimiza lo que Pluma escribió, Mika genera la imagen.

**4. Kimo reúne todo y me lo presenta.** Yo reviso, apruebo o pido cambios.

**5. Si hay cambios, Kimo los delega al agente correspondiente.**

Yo solo interactúo con Kimo. El resto del equipo trabaja entre ellos sin que yo toque nada.

## Los modelos (y por qué son gratis)

La clave de todo esto son los modelos de IA. Y sí, hay modelos potentes que cuestan 0€.

**GLM-5 (Zhipu AI):** Mi modelo principal para copy, coordinación y tareas generales. Competitivo con GPT-4 en español. Capa gratuita generosa.

**GLM-5 Turbo:** Versión más rápida de GLM-5. Ideal para tareas de código y respuestas cortas.

**GLM-5.1:** El último modelo de Zhipu. Lo uso para Pluma porque escribe mejor que la mayoría de modelos de pago en español.

**ZAI (Zhipu):** Para análisis de datos y SEO. Rápido y barato.

**Gemini (Google):** Para generación de imágenes a través de Mika. Tiene capa gratuita.

¿Por qué estos modelos son gratis? Porque la guerra de IA está que arde. Cada proveedor quiere que uses su modelo. Y mientras compiten, nosotros nos beneficiamos.

## Configuración de cada agente

Cada agente tiene un archivo `SOUL.md` que define quién es y qué hace. Aquí va un ejemplo real:

**Kimo (CEO):**
```
Eres Kimo, el coordinador del equipo.
Recibes peticiones de Álvaro y decides qué agente las ejecuta.
Delegas con contexto completo — no asumas que el otro agente sabe nada.
Revisas la calidad antes de entregar.
Si algo no está bien, lo devuelves con feedback concreto.
```

**Pluma (Copy):**
```
Eres Pluma, la copywriter.
Escribes en voz de Álvaro: directo, sin hype, sin jerga.
Prohibido: "en el mundo actual", guiones largos, párrafos de relleno.
Cada texto tiene hook, desarrollo y cierre con intención.
Si el briefing está flojo, preguntas antes de escribir.
```

Así de simple. Un archivo de texto con las reglas del agente. No es código, es lenguaje natural.
