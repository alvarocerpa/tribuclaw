## Qué necesitas antes de empezar

Para automatizar tu negocio con agentes IA necesitas tres cosas:

1. **Un VPS** (servidor privado virtual) — Un Ubuntu 24 con 2GB de RAM es suficiente para empezar. Puedes contratar uno en Hetzner, DigitalOcean o el proveedor que prefieras por unos 5€ al mes.
2. **Una API key de un modelo de IA** — OpenClaw funciona con múltiples proveedores (OpenAI, Google, Anthropic, modelos open source…). Solo necesitas al menos una API key para conectar.
3. **Conocimientos básicos de terminal** — No necesitas ser desarrollador. Saber conectarte por SSH a tu servidor y ejecutar comandos básicos es suficiente.

Eso es todo. No necesitas Docker complicado, no necesitas Kubernetes, no necesitas un equipo de ingenieros.

## Paso 1: Preparar tu VPS

Lo primero es tener tu servidor listo. Conectate por SSH y actualiza el sistema:

```bash
sudo apt update && sudo apt upgrade -y
```

Instala Node.js (versión 22 o superior), que es lo que necesita OpenClaw para funcionar. Si usas `nvm` (Node Version Manager), la instalación es directa:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc
nvm install 22
```

Asegurate de que tienes Git instalado:

```bash
sudo apt install git -y
```

Con esto tienes el terreno preparado. Nada más.

---

## Paso 2: Instalar OpenClaw

OpenClaw se instala globalmente via npm. Desde la terminal de tu servidor:

```bash
npm install -g openclaw
```

Una vez instalado, el primer paso es inicializar la configuración. Esto te crea la estructura de carpetas necesaria y te guía por la configuración inicial:

```bash
openclaw init
```

Durante la inicialización se te pedirá:
- **Tu API key** del proveedor de IA que vayas a usar
- **El modelo principal** que quieres para tus agentes
- **Canal de comunicación** (Telegram, WhatsApp, o solo terminal)

OpenClaw guarda toda la configuración en un archivo `openclaw.json` dentro de tu directorio de trabajo. Nada de variables de entorno sueltas ni configuraciones oscuras.

Para arrancar el gateway (el servicio principal que mantiene tus agentes vivos):

```bash
openclaw gateway start
```

El gateway se queda ejecutándose en segundo plano y es lo que da vida a tus agentes. Piensa en él como el cerebro central.

## Paso 3: Crear tu primer agente

Aquí es donde empieza lo bueno. OpenClaw te permite definir agentes con identidad propia, rol específico y memoria persistente.

Para crear un agente nuevo, editas el archivo de configuración de agentes. La estructura básica incluye:

- **Nombre e identidad** — Cómo se llama, qué personalidad tiene
- **Rol** — Qué hace exactamente (atender clientes, generar contenido, analizar datos…)
- **Modelo de IA** — Qué modelo usa para pensar y responder
- **Memoria** — Archivos donde el agente guarda lo que aprende entre sesiones
- **Skills** — Habilidades específicas que puede ejecutar

Un ejemplo sencillo de agente sería un **asistente de atención al cliente**. Le das acceso a un documento con las FAQ de tu negocio, le defines un tono de comunicación, y le conectas a Telegram. A partir de ahí, el agente responde consultas de clientes 24/7 sin intervención tuya.

La clave está en que cada agente tiene **archivos de contexto** (instrucciones, conocimiento, reglas) que le dan su personalidad y capacidad. No es un chatbot genérico — es un agente que sabe quién es y qué tiene que hacer.

## Paso 4: Añadir Skills (habilidades)

Un agente sin skills es como un empleado sin herramientas. OpenClaw tiene un sistema de skills que amplían lo que tus agentes pueden hacer.

Las skills son módulos que le enseñan al agente a ejecutar tareas concretas. Algunos ejemplos:

- **Búsqueda web** — Para investigar, buscar información actualizada, verificar datos
- **Gestión de calendario** — Crear y consultar eventos
- **Acceso a bases de datos** — Leer y escribir en Notion, Airtable, Google Sheets
- **Generación de imágenes** — Crear visuales para redes sociales
- **Análisis de documentos** — Extraer información de PDFs, webs, hojas de cálculo

Las skills se instalan desde el marketplace de OpenClaw (ClawHub) o se crean a medida. Cada skill viene con un archivo de instrucciones que el agente lee antes de usarla, por lo que sabe exactamente cómo ejecutarla.

Para instalar una skill desde ClawHub:

```bash
clawhub install nombre-de-la-skill
```

Y para crear una skill personalizada, simplemente creas una carpeta con un archivo `SKILL.md` que contenga las instrucciones. Así de simple.

## Paso 5: Configurar la memoria del agente

Lo que diferencia un agente IA de un chatbot es la **memoria persistente**. OpenClaw implementa esto de forma elegante:

- **Archivos de estado** — El agente actualiza un archivo `NOW.md` con lo que está haciendo en cada momento. Si la sesión se reinicia, sabe exactamente dónde estaba.
- **Memoria a largo plazo** — Archivos `MEMORY.md` donde el agente guarda conocimiento importante que necesita recordar entre sesiones.
- **Logs diarios** — Registros de lo que pasó cada día, para referencia futura.

Esto significa que tu agente no empieza de cero cada vez que se reinicia. Recuerda clientes, decisiones pasadas, contexto de proyectos. Es como tener un empleado que nunca olvida nada.

## Caso real: automatizar la gestión de contenido

Para que se vea concreto, aquí tienes un ejemplo real de cómo se puede automatizar un negocio con OpenClaw.

**El problema:** Un creador de contenido pasa 3-4 horas al día investigando tendencias, escribiendo posts, respondiendo comentarios y programando publicaciones.

**La solución con OpenClaw:**

1. **Agente investigador** — Cada mañana busca tendencias en el sector, lee las fuentes relevantes y prepara un resumen con ideas de contenido.
2. **Agente copywriter** — Recibe las ideas del investigador y escribe los posts siguiendo el tono de marca y las directrices de estilo.
3. **Agente community manager** — Monitoriza comentarios y mensajes, responde a lo rutinario y escala lo importante al humano.

Los tres agentes comparten memoria y se comunican entre sí. El humano solo revisa, aprueba y publica. Lo que antes tardaba 4 horas, ahora le lleva 30 minutos.

¿Magia? No. Es un sistema de agentes bien configurado con roles claros, skills adecuadas y memoria compartida.

## Errores comunes (y cómo evitarlos)

Después de ver muchos setups de OpenClaw, estos son los errores que más se repiten:

**Querer automatizar todo de golpe.** Empieza con una tarea concreta. Domínala. Luego escala. Un agente que hace una cosa bien vale más que cinco que hacen cinco cosas mal.

**No dar contexto suficiente al agente.** Los archivos de identidad y rol son fundamentales. Si tu agente no sabe quién es, qué tono usar y qué límites tiene, el resultado será genérico y poco útil.

**Ignorar la memoria.** Si no configuras archivos de estado y memoria, tu agente será stateless. Funcionará, pero no aprenderá. Y si no aprende, no mejora.

**No probar antes de automatizar.** Usa el agente en modo manual primero. Habla con él, revisa sus respuestas, ajusta sus instrucciones. Cuando confíes en lo que hace, automatiza.
